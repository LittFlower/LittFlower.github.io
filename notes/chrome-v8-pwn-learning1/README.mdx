<h2 id class="scroll-target"><a href="#"><code class="hash inline-code">#</code>简介</a></h2>
<blockquote>
<p>V8 是一个由 Google 开发的开源 JavaScript 引擎，用于 Google Chrome 及 Chromium 中，项目以V8发动机其命名。此项目由Lars Bak主导开发。
——摘自维基百科</p>
</blockquote>
<p>JavaScript 是解释语言，需要先翻译成字节码后在 VM 上运行。V8 中实现了一个 VM。出于性能考虑，目前的引擎普遍采用一种叫做 Just-in-time (JIT) 的编译技术，V8 也是。JIT 的思想在于，如果一段代码反复执行，那么将其编译成机器代码运行，会比每次都解释要快得多。</p>
<p>v8 这个引擎编译出来的二进制文件为 d8，我们 Pwner 攻击的主要目标一般就是这个。</p>
<p>v8 的开源仓库在<a href="https://chromium.googlesource.com/v8/v8/" title="https://chromium.googlesource.com/v8/v8/" target="_blank" rel="noopener">这里</a></p>
<h2 id class="scroll-target"><a href="#"><code class="hash inline-code">#</code>环境搭建</a></h2>
<h3 id class="scroll-target"><a href="#"><code class="hash inline-code">#</code>最新版本</a></h3>
<p>首先，由于需要大量在 google 的网站和 github 上下载代码，所以需要配置好代理，这里略过不表。</p>
<p>随后，先安装 depot_tools，arch linux 可以使用 aur 仓库里打包好的软件包，其他发行版可能需要 clone 项目<a href="https://chromium.googlesource.com/chromium/tools/depot_tools.git" title="https://chromium.googlesource.com/chromium/tools/depot_tools.git" target="_blank" rel="noopener">仓库</a>。这个工具包是用来管理 v8 代码的。</p>
<p>然后安装 ninja，arch linux 的 extra 仓库里有这个包，直接下就行，其他发行版可以自己 <a href="https://github.com/ninja-build/ninja.git" title="https://github.com/ninja-build/ninja.git" target="_blank" rel="noopener">clone</a> 编译。</p>
<p>然后编译 v8 代码即可，v8 的编译结果可以有两种，release 或者 debug 版本，我们一般使用 debug 版本方便调试。</p>
