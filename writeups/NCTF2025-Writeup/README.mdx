<p>注意到：</p>
<ul>
<li><code class="inline-code">read_int</code> 的返回值为 <code class="inline-code">int</code>，<code class="inline-code">size</code> 是 <code class="inline-code">unsigned int</code>，输入 <code class="inline-code">-1</code> 的话返回值就是 <code class="inline-code">0xffffffff</code></li>
<li><code class="inline-code">malloc()</code> 的参数类型是 <code class="inline-code">size_t</code>，也就是 <code class="inline-code">unsigned int</code></li>
<li><code class="inline-code">chunk[idx]->size</code> 的类型是 <code class="inline-code">__int64</code></li>
<li><code class="inline-code">malloc(0)</code> 等价于 <code class="inline-code">malloc(0x10)</code></li>
</ul>
<p>所以如果输入 <code class="inline-code">-1</code>，程序会执行 <code class="inline-code">malloc(0)</code>（因为 <code class="inline-code">0xffffffff + 1</code> 等于 <code class="inline-code">0x1 00000000</code>造成溢出截断)，但是 <code class="inline-code">chunk[idx]->size</code> 会记录 <code class="inline-code">0x1 00000000</code>，这就造成了无限的堆溢出。</p>
<p>有了堆溢出之后相当于有了 UAF，可以打 tcache dup + posioning + environ 泄漏栈地址 + rop。</p>
