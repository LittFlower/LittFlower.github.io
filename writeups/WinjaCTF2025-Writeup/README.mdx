<p>然后 func1 和 func2 是两个数组，程序逻辑大概是 %s 读入一段任意长的数据，前 4 字节会把从 <code class="inline-code">func1</code> 到 <code class="inline-code">xor</code> （不包括 xor）的所有字节全部循环异或一遍，然后执行 <code class="inline-code">func1</code>，在用 <code class="inline-code">func2</code> 和 <code class="inline-code">rbp-0x30</code> 处的数据循环异或，最后执行 <code class="inline-code">func2</code>。</p>
<p>注意，我这里并没有说 <code class="inline-code">v4</code> 而是说 <code class="inline-code">rbp-0x30</code> 就是暗示可以通过调 rbp 的位置来做到把 <code class="inline-code">func2</code> 变成 shellcode getshell。</p>
<p>逆天的来了，题目并没有告诉这个程序的可执行版本，也就是说我一开始用我的宿主机（arch linux，glibc 3.41）跑这个题的时候，<code class="inline-code">r12</code> 寄存器上是 <code class="inline-code">1</code>，其他寄存器也没有保留任何栈地址，私以为这种和版本强相关的程序应该给个 libc 或者在题目描述里说一下，但是这个逆天比赛根本没提，所以我一开始死活打不出来。</p>
