<h2 id="dbfs" class="scroll-target"><a href="#dbfs"><code class="hash inline-code">#</code>dbfs</a></h2>
<p>出题人给了源码，实现了一个简单的文件系统：</p>
<ul>
<li><code class="inline-code">SET &lt;name> &lt;type> [value]</code>: 创建键或目录</li>
<li><code class="inline-code">GET &lt;name></code>: 获取键值或列出目录内容</li>
<li><code class="inline-code">DELETE &lt;name></code>: 删除键或目录</li>
<li><code class="inline-code">INFO &lt;name></code>: 显示键或目录的详细信息</li>
<li><code class="inline-code">CHDIR &lt;name></code>: 切换当前目录</li>
</ul>
<p>在 <code class="inline-code">handler_info</code> 函数里，下面这段代码存在栈溢出漏洞：</p>
<div class="code-snippet"><pre class="code-snippet__pre"><code class="language-c language-c"><span style="--shiki-light:#622CBC;--shiki-dark:#DBB7FF">getcwd</span><span style="--shiki-light:#0349b4;--shiki-dark:#91cbff">(</span><span style="--shiki-light:#0E1116;--shiki-dark:#F0F3F6">full_path, </span><span style="--shiki-light:#A0111F;--shiki-dark:#FF9492">sizeof</span><span style="--shiki-light:#055d20;--shiki-dark:#4ae168">(</span><span style="--shiki-light:#0E1116;--shiki-dark:#F0F3F6">full_path</span><span style="--shiki-light:#055d20;--shiki-dark:#4ae168">)</span><span style="--shiki-light:#0349b4;--shiki-dark:#91cbff">)</span><span style="--shiki-light:#0E1116;--shiki-dark:#F0F3F6">;</span><br><span style="--shiki-light:#622CBC;--shiki-dark:#DBB7FF">strcat</span><span style="--shiki-light:#0349b4;--shiki-dark:#91cbff">(</span><span style="--shiki-light:#0E1116;--shiki-dark:#F0F3F6">full_path, </span><span style="--shiki-light:#032563;--shiki-dark:#ADDCFF">"/"</span><span style="--shiki-light:#0349b4;--shiki-dark:#91cbff">)</span><span style="--shiki-light:#0E1116;--shiki-dark:#F0F3F6">;</span><br><span style="--shiki-light:#622CBC;--shiki-dark:#DBB7FF">strcat</span><span style="--shiki-light:#0349b4;--shiki-dark:#91cbff">(</span><span style="--shiki-light:#0E1116;--shiki-dark:#F0F3F6">full_path, name</span><span style="--shiki-light:#0349b4;--shiki-dark:#91cbff">)</span><span style="--shiki-light:#0E1116;--shiki-dark:#F0F3F6">;</span><br></code></pre></div>
<p><code class="inline-code">strcat</code> 没有做长度检查。但是问题是 <code class="inline-code">strcat</code> 默认会用 <code class="inline-code">\x00</code> 结尾，导致无法利用下面的 <code class="inline-code">printf</code> 泄漏地址，所以尝试找别的洞。</p>
<p><img loading="lazy" decoding="async" fetchpriority="auto" src="https://pic1.imgdb.cn/item/693680db2a4ee13cb950938e.png" alt="hex2bytes"/></p>
<p>审计了一会发现 <code class="inline-code">hex2bytes</code> 函数里，如果输入非法的 hex 这里解析会失败，导致堆块里残留的地址不会被覆盖掉，所以风水一下搞个 libc 地址就可以了。</p>
<p>然后在打 rop 的时候会发现 payload 里不能出现 \x00，不然会截断导致报错。写个栈迁移到堆上即可。</p>

<h2 id="shortcut" class="scroll-target"><a href="#shortcut"><code class="hash inline-code">#</code>shortcut</a></h2>
<p>牢了最久的题。。</p>
<ul>
<li><code class="inline-code">add</code> <code class="inline-code">delete</code> <code class="inline-code">run</code> 时进行 <code class="inline-code">overlap</code> 检查</li>
<li><code class="inline-code">add</code> 最多 8 个，大小固定 0x20，不限机会，但 <code class="inline-code">run</code> 之后不可用，必须先 <code class="inline-code">reload</code></li>
<li><code class="inline-code">delete</code> 功能释放所有堆块，最后一个槽位指针不置 0，两次机会</li>
<li><code class="inline-code">run</code> 可以显示堆块内容并释放 img chunk，两次机会</li>
<li><code class="inline-code">reload</code> 分配 img chunk，两次机会</li>
<li><code class="inline-code">rename</code> 分配 0x30，并可控 0x10，一次机会</li>
</ul>
